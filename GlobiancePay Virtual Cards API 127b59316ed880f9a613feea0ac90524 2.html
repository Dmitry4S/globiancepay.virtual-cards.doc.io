GlobiancePay Virtual Cards API

Base URL

The base URL for all API requests is:

https://uat-card-issuer.globiancepay.io/api

1. API security access

1.1 Overview

The open API has 3 security measures to control access and ensure data security: IP whitelist, user token authentication, and AES encryption of data.
Before formal integration, users need to provide the IP addresses from which they will access the API. We will then provide users with the necessary access token and AES key for accessing the API securely.

1.2 Encryption

  1. Data encryption uses AES in CBC mode with PKCS7 padding.
  1. The IV used for encryption is fixed to the hexadecimal "\x00\x01\x02\x03\x04\x05\x06\x07\x08\x09\x0A\x0B\x0C\x0D\x0E\x0F".
  1. After encryption, encode using base64.

1.3 Example of data transmission

  1. Convert the business parameters to JSON format ready for encryption, for example: {"test":"test"}.
  1. Perform AES encryption, if the key is "s3429EjAU9WK7cm4", and encode using Base64. The resulting ciphertext is "CVW0KWKkzXmWIxqBVgpJbw==". You can verify the encryption using the testing tool at https://gchq.github.io/CyberChef.
  1. Place the encrypted string into the "data" field of the JSON to be transmitted. { "data": "CVW0KWKkzXmWIxqBVgpJbw==” }
  1. When making the HTTP request, add the "Autoriisation" header and assign the access token provided to you.
  1. Use HTTP POST method to send the entire json in the third step through HTTP body.
  1. The returned data is in JSON format, and the "data" field is encrypted using the same method. Decrypting it will yield the original business data.

Example of the returned data

{
    "code": 1,
    "msg": "card issue success",
    "data": "bfKzWiGNU92CzWV3rhUc"
}

Returns a list of all cards.

Parameters

POST /api/cards/card-issue

Issue the card.

Parameters

  • productCode : Code of product.
  • currency : Currency
  • cardType : Type of card: single_issue | multi_issue
  • amt : Amount
  • expdate : Expiration date
  • remark : Remark

Example

Request:

POST /api/cards/card-issue

{
  "productCode": "E0000001",
  "currency": "USD",
  "cardType": "single_issue",
  "amt": 100,
  "expdate": "2025-11-30",
  "remark": "Card Remark"
}

Errors

This API uses the following error codes:

  • 400 Bad Request: The request was malformed or missing required parameters.
  • 401 Unauthorized: The API key provided was invalid or missing.
  • 404 Not Found: The requested resource was not found.
  • 500 Internal Server Error: An unexpected error occurred on the server.
